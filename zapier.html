Listed below is a discussion of 3 alternative designs for these pages. I decided to build them using the current CSS/layout to make them feel familiar and to minimize rework using a dedicated design/wireframe tool.

Design 1 - Social Confidence

By the time a user has reached a page for an app in the Zapbook they have probably already been exposed to the "500+ apps" marketing. The frame of mind at this point is to narrow the scope to find a particular zap. With >100 options to scroll through this can be overwhelming! Plus, there is no indication currently why these are "recommended zaps". By showing that other users have used these same zaps will provide the user with confidence to either try one out or continue exploring.

Hypothesis: Currently, the right button for "1-5 (of 100)" isn't clicked all that often resulting in low click-through for recommended zaps. With this design, seeing that zaps have been used by others will provide confidence and result in more click-through.

Metrics: % click and conversion via recommended apps

Design 2 - Take Action!

When showing the page to a friend they remarked, "I didn't even know you could scroll down." This design attempts to make the most of the users attention the moment they reach the page to get them invested through action. My attempt with the design is to proactively prompt thinking in the user as to how they might combine a specific app. It would also eliminate much of the various other sections to focus on building a zap. The more the user interacts with the sight (without hitting a wall of frustration) the more likely they are to understand the value and sign-up.

Hypothesis: Currently, users mostly abandon the page or scroll down without clicking the first CTA. By creating obvious interactive elements the user is more likely to find a matched app and follow through with zap creation.

Metrics: % selection of an app and conversion through "continue".

Design 3 - Human Touch

Users may not yet have the confidence that "even I can create a zap." A reference will help to build that confidence and also validate the value of a zap. The particular reference can be rotated (using existing segmentation I found in the page source of lawyer, journalist, etc) based on the user to build as close a connection as possible.

Hypothesis: Currently, users mostly abandon the page or scroll down without clicking the first CTA. By showing a compelling reference the equation is completed and the user feels confident to sign-up.

Metrics: % clicking "sign-up" right away!
